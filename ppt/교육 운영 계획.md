# AI 활용 2차 TF 자체 교육 운영 계획

> '26. 5. | 정보시스템실

---

## 1. 개요

**1.1 목적**

- 2차 TF 위원의 AI 활용 원리 이해 및 본인 업무에 적용 가능한 자동화 도구 제작 역량 배양
- **각 위원이 본인 업무에서 직접 사용할 도구 1개를 교육 기간 내 완성**하는 것을 핵심 목표로 함

**1.2 기간 및 회차**

- 기간 : 2026. 5. 26. (화) ~ 6. 9. (화)
- 총 **3회 / 회당 3시간 / 합계 9시간**

**1.3 대상**

- 관리업무 개선 2차 TF 위원 **10명** (1차 TF 위원 제외, 관리부서 신청자)

**1.4 진행**

- 정보시스템실 자체 진행 (라이브 시연 + 핸드아웃 방식)

---

## 2. 운영 방향

가. **압축 + 자체교육 전환**

- 기존 4회차 외부 강사 계획 → **3회차 정보시스템실 자체 교육**으로 변경
- 1차 TF 교육(12시간) 핵심을 9시간으로 압축
- 4회차에 편성됐던 1:1 멘토링이 빠진 만큼, **회차 중·회차 간 과제에 도구 완성 압력을 분산 배치**

나. **'도구 1개 완성' 트랙**

- 매 회차마다 도구 제작을 향한 점진적 산출물을 요구하여, 3회차에 도구를 *처음* 만들기 시작하는 게 아니라 **1·2회차에 이미 컨셉·프롬프트·입출력이 모두 결정된 상태에서 코딩만 남도록 설계**

| 회차       | 도구 트랙 산출물                      |
| -------- | ------------------------------ |
| 1회차      | 자동화 후보 3개 + 우선순위               |
| 1→2회차 과제 | 후보 3개 입출력 ChatGPT 시뮬레이션        |
| 2회차      | 도구 컨셉 1개 확정 (화면 스케치 + 핵심 프롬프트) |
| 2→3회차 과제 | 본인 실제 업무 데이터로 도구 입출력 완전 시뮬레이션  |
| 3회차      | HTML 도구 v1 완성 + 시연             |

다. **실습 비중 확대**

- 1차 교육 피드백 반영 : 이론 최소화, 실습 70% 이상
- 매 회차마다 본인 업무 데이터를 직접 다루는 시간 확보

## 3. 회차별 주요 내용

### 3.1 1회차 — LLM 원리 및 인터뷰 프롬프트 (5. 26. 화)

가. 교육 목표

- LLM이 언어를 이해·생성하는 원리(자동완성, 의미공간, 작업기억) 이해
- 본인 업무 맥락을 AI에게 전달하는 능력 확보

나. 주요 내용
&nbsp;&nbsp;1) LLM 핵심 개념 : 거대언어모델을 단순 검색기가 아닌 **'확률적 자동완성 도구'** 로 정의, 의미공간·작업기억 개념 학습
&nbsp;&nbsp;2) 맥락(Context)의 중요성 : 더미 데이터를 활용한 실습을 통해 맥락 제공 유무에 따른 결과 차이 체험
&nbsp;&nbsp;3) 인터뷰 프롬프트 실습 : 강사 제공 '인터뷰 프롬프트'를 활용하여 본인 업무 맥락을 구체화하는 문서 작성
&nbsp;&nbsp;4) **자동화 후보 발굴** : 본인 업무 중 반복·정형 작업 3개 후보 도출

다. 회차 성과물

- **업무 맥락 정의서** (인터뷰 프롬프트 결과)
- **자동화 후보 3개 + 우선순위**

라. 1→2회차 과제

- 후보 3개 각각에 대해 ChatGPT/Gemini로 *"어떤 입력 → 어떤 작업 → 어떤 출력"* 을 텍스트로 시뮬레이션 (실제 도구화 가능 여부 사전 판단)

### 3.2 2회차 — 출력 형식 다양화 및 도구 컨셉 확정 (6. 2. 화)

가. 교육 목표

- 다양한 출력 포맷 활용 능력 및 프롬프트 3요소 구조화
- **만들 도구 1개를 확정**하고 핵심 프롬프트·입출력 정의

나. 주요 내용
&nbsp;&nbsp;1) 출력 형식의 확장 : Markdown, HTML, Mermaid 코드 등으로 변환하여 도표·다이어그램으로 시각화 실습
&nbsp;&nbsp;2) 업무 흐름도 작성 : 1회차 성과물(업무 맥락 문서)을 바탕으로 본인 업무 프로세스를 플로우차트로 시각화
&nbsp;&nbsp;3) 프롬프트 3요소 구조화 : 입력(Input) / 작업 지시(Instruction) / 출력 형식(Output) 명확히 구분
&nbsp;&nbsp;4) 반복(Iterative) 개선 : 실행 결과 분석·수정을 반복하며 최적 답변을 도출
&nbsp;&nbsp;5) **도구 컨셉 확정 (회차 후반 30분)** : 후보 3개 중 1개 선택, 화면 스케치 1장, 핵심 프롬프트 초안

다. 회차 성과물

- 본인 업무 프로세스 흐름도
- **확정된 도구 컨셉 1건** (화면 스케치 + 핵심 프롬프트 초안)

라. 2→3회차 과제

- 도구 입출력을 ChatGPT/Gemini로 *본인 실제 업무 데이터를 사용해* 완전히 시뮬레이션 (3회차에는 코딩만 남도록)

### 3.3 3회차 — 바이브 코딩 및 도구 완성 (6. 9. 화)

가. 교육 목표

- AI를 도구 제작자(Toolmaker)로 활용하여 **본인 업무 도구 완성**

나. 주요 내용
&nbsp;&nbsp;1) 바이브 코딩 개요 : AI에게 HTML/JS 코딩을 지시하는 기법, 단일 파일 도구로 한정
&nbsp;&nbsp;2) 트러블슈팅 : 결과가 의도와 다를 때의 상황을 타개하는 **'인터뷰 프롬프트'** 활용법
&nbsp;&nbsp;3) **개별 도구 제작 실습** (회차의 60% 이상) : 사전 정의된 컨셉·프롬프트를 코드로 구현
&nbsp;&nbsp;4) **결과 발표** (회차 마지막 30분) : 각자 만든 도구를 2~3분씩 시연

다. 회차 성과물

- **HTML 기반 개인 업무 도구 v1** (브라우저에서 바로 실행 가능)
- 트러블슈팅 노트

## 4. 운영 방식

가. **자료 형식**

- 회차별 **강사 진행 노트**(타임라인·시연 시나리오) + **수강생 핸드아웃**(1~2장 요약·실습 칸)
- 자료는 SharePoint 또는 메일 사전 배포

나. **실습 도구**

- ChatGPT Plus 또는 Gemini Advanced (개인별 택 1, 기존 운영안 유지)
- 도구 결과물은 **브라우저에서 바로 실행되는 HTML/JS 단일 파일**로 한정 (사내 환경 제약 회피)

다. **운영 원칙**

- 이론 최소화, **실습 중심** 운영 (1차 교육 피드백 반영)
- 매 회차 시작 시 전 회차 내용 **복습 5분**
- 회차 간 과제를 통해 실습 내용이 본인 업무로 이어지도록 설계
- 교육 종료 후에도 정보시스템실을 통해 **상시 문의 창구** 운영
